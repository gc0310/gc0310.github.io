Final Game: Feed Tubbs!

Feed Tubbs was inspired by an app called ‘Neko Atsume’, which allows the user to feed cats. This is an adaptation of the clicker game we did in class as I thought I could improve on the previous game by adding new elements to the game. A live game time display, including a live score board was added so that users can see their progress.

So I designed the game where objects would be moving around the board, and the player would have to click on the target object in order to score points. There are two types of objects the player could click on, one was the ‘Sushi’ - which earned the player points, while the other was ‘Canned Food’ - which deducted points from the player’s overall score. The ‘Canned Food’ was added to make it more challenging for the user to click on the ’Sushi’. The objective of the game was to get players to score as high as possible.

Instead of using Javascript to create squares and circles as my target objects, I used images which gave me more control over the design, as well as to make the game look more consistent as I could source for images on my own. The images were obtained via Google, and I used Photoshop and Illustrator respectively to edit them for it to fit the game. I used an image for my start button, which I gave an event listener to so that when it is clicked, it will initialise the start function. The start function included a prompt which invited the user to type in their desired difficulty level. If the user typed in any other number than the ones given (1, 2, 3), the game will be set to the lowest level.

The if conditional was used to set the various difficulty levels. As each difficulty level increased, the speed at which the target object was increased, along with the number of distracting objects. Each moving object had a ‘move’ function, which was called every 1/100th of a second. This allowed each object to be moved across the board smoothly.

On the front page, I made the two icons rotate to make the game look more interactive. I adapted what we learnt from the ‘Draw Smiley Face’ assignment, which required us to animate the path from a set point. For this assignment, I used the animation to rotate the images, and I made a function for each image. This function was called every 2 seconds to make the animation continue forever.

I made 6 individual ‘Canned Food’ objects, and this allowed me to initialize their rates individually. Each can had their own ‘moveCan’ function, which was written with ‘if’ conditions so that the object wouldn’t move out of the board.

I created a function ‘randInt’, which used the ‘Math.random()’ Javascript function to generate a random number. This was used to give the objects random X and Y coordinates. This also ensured that each object had a distinct path from the other and there were no overlaps.

The game was intended to run for 15 seconds, however, due to the delay in the live timer display, the user will only see it when it starts counting down from 14. Hence, I set the game to run for 16 seconds, which gives users a clearer view on how long they’ve left in the game.

I got my sounds from YouTube (https://www.youtube.com/watch?v=OmSQuxbDwVE), where I downloaded the video and converted it to mp3. I got my sound effects from this website http://taira-komori.jpn.org/game01en.html, and when a click was made on the ‘Sushi’ (the correct target object), a sound would be made to tell the user it was registered. In contrast, if a click was made on the ‘Canned Food’ (the wrong target object), a bloop sound would be made to signify that the score was deducted.

In retrospect, I could have created a function which made the objects move in a different direction when clicked. This would’ve made the game slightly more challenging, and more interesting for the player as well. I could have created a function which changed the target image when clicked using arrays. However, I encountered a lot of difficulty in doing so, and I could not get it to move in a similar manner as static images.
